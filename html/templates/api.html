Foreword

The entirety of the Barbs API is stored externally here. If you change something on the “API Scripts for Barbs” page on Roll20, you should let me know, or else I will inevitably overwrite your change.


There is a ton of hard-coded information in the Barbs API. This means that every item must be manually created and added to a master list of items. Every ability must be manually implemented for the API to handle it properly. If you want to add something yourself, you may, but see the note above. If you want a particular item or ability added and don’t want to do it yourself, talk to me.

Character Sheet

As in a standard DnD 5e campaign, there is a “Character Sheet” tab on your character’s page on Roll20. You should fill out:

  • Attributes – These are 1-1 with column D on your Google Sheets character sheet.
  • Skills – Specify the proper name of each skill and the number of AP you have put into each ability. The sheet will update the rank and bonus for each skill when you add points. If it does not update, you likely have the skill name wrong.
  • Class Abilities – The class name and ability name are required for each ability. Additionally include the tier level (0-4) if you want your level on the Characteristics page to be accurate.
  • Equipment – Use the real, full name of the item. The parameter lines can be left blank.

The “Stats” page shows your stats based solely on your attributes. If you roll one of these stats, however, your equipped items will be factored into the roll. Effects from class abilities do not factor in currently - they will eventually.

General

Identification

The API generally identifies you by who you are speaking as in Roll20 chat. If Roll20 profile name or character name changes, the API may no longer work for you.


Ability Parameters

Many abilities require “parameters,” extra information that you must specify for the ability to work correctly. Parameters for an ability are specified in the line below the ability on your character sheet. Multiple parameters for one ability can be specified, and are separated by semicolons. Parameters usually take the form “keyword value(s)”. An example set of parameters may look like:

                distance 100; spotting 8


Persistent Effects

Persistent effects, like a damage multiplying buff that lasts 1 minute, will be included in rolls automatically. The effect will be removed automatically when the turn order progresses through the required number of rounds. Other effects, like empowers, will be removed after they are applied to an attack. If you want to manually end a persistent effect, use the following command.

  • !barbs remove_effect <character name>;<effect name>

Note the semicolon. A character’s first name is sufficient. The effect name is usually the name of the ability that generated the effect. The API will send a message if it found and removed the effect.


Crit Rolls

Crit rolls appear a little oddly. A negative crit roll is failure. Zero and above is a success. If a crit roll is a success, the entire crit roll section in chat will be outlined in green. If you hover over the crit roll, the calculation is “what you rolled” minus “the threshold you needed to meet for this to be a crit”. The crit threshold is adjusted based on your attributes, items, and abilities’ effects.

Items

The Barbs API scripts contain a big list of items. If you have item X equipped and it is not in this list, its effects will not be applied to any roll. Theoretically, eventually, I will make some kind of form that makes it relatively easy to input an item’s characteristics and output a new entry for this list. Such a form does not exist yet though. If you want to look at other items in the list as examples and then make your own, that would be a massive help. Again, let me know if you add something though, or I will overwrite it sooner or later.


You can do an auto-attack with an item by clicking the roll button on the “Equipment” tab of your character sheet. If other abilities you have used require you to specify parameters, you can do so in the boxes below the main hand and offhand slots. Parameters here behave the same as parameters for classes.

Class Abilities

This section documents how to use each class ability via the character sheet.


Classes and abilities that are “not implemented” will be eventually. For now if you try to use one of these abilities, the API will say it is unknown. If there are no notes for an ability, the API does as the ability describes. There are, however, many abilities that do things the API will not handle. Action economy, for example, is not tracked by the API. For abilities like this, the API will just print the ability’s description. It is on the players to deal with whatever the ability does in these cases.

Extra

Nothing is ever perfect. If there is some damage or multiplier that you need to include when using an ability that isn’t handled by the API, you can use the “misc” parameters.

  • misc_multiplier <multiplier value> <damage type> <source>
    • Multiplier values should be specified as decimal (0.2) rather than percentage (20%). You do not need to add 1 - a 20% increase is a 0.2 multiplier.
    • Damage types here are: physical, fire, water, earth, air, ice, lightning, light, dark, healing, all_magic, all
    • The source is the origin of the multiplier. Items, for example, use source “self”. If the source is another character, use their character’s full name.
    • If you want 20% increased air damage, for example, you could add "misc_multiplier 0.2 air self".
  • misc_damage <damage dice> <damage type>
    • The damage dice should be something like “5d8”. You can also put a constant.
    • Damage types here are: physical, fire, water, earth, air, ice, lightning, light, dark, healing
